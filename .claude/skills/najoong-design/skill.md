# 나중(Najoong) 디자인 가이드

## 디자인 방향

Claude(claude.ai) UI의 톤을 기반으로 한다. 눈이 편안한 따뜻한 오프화이트 배경 위에,
나무·점토를 연상시키는 웜 클레이 색을 포인트로 쓴다. 서비스 컨셉인 "서랍"과
어울리는 따뜻하고 차분한 분위기를 지향한다.

핵심 원칙:
- 순백(#FFFFFF)을 배경 전체에 쓰지 않는다. 배경은 살짝 크림빛 도는 오프화이트,
  카드·입력창만 순백으로 올려 층(depth)을 만든다.
- 포인트 색은 아껴 쓴다. 저장 버튼, 활성화된 항목, 서랍 일러스트 등 "강조"에만.
  나머지는 전부 화이트 + 웜 그레이로 처리한다.
- 화려한 효과보다 여백·정렬·타이포로 완성도를 낸다.

## 색상

두 계열만 쓴다: 배경 계열(오프화이트 + 순백)과 포인트 계열(웜 클레이).
텍스트는 색으로 강조하지 않고 웜 다크/그레이로 통일한다.

| 역할 | 값 | 용도 |
|------|------|------|
| 페이지 배경 | `#FAF9F6` | 전체 배경. 살짝 크림빛 오프화이트 |
| 카드·입력창 배경 | `#FFFFFF` | 순백. 배경과 대비로 떠 보이게 |
| 포인트 (클레이) | `#BE7250` | 저장 버튼, 활성 항목, 서랍, 강조 요소 |
| 포인트 hover | `#A85E3E` | 포인트 요소 hover 시 살짝 어둡게 |
| 포인트 배경 틴트 | `#F5EBE4` | 활성 항목의 옅은 배경, 뱃지 배경 |
| 본문 텍스트 | `#2B2925` | 순검정 대신 살짝 갈색 도는 다크 |
| 보조 텍스트 | `#8B857C` | 설명, 도메인, placeholder 등 |
| 테두리 | `#EDE9E3` | 카드·입력창 경계. 웜 톤 얇은 선 |

규칙:
- 새로운 원색·고채도 색을 임의로 추가하지 않는다.
- 빨강·초록 같은 상태색은 꼭 필요한 곳(삭제, 오류)에만 최소한으로.
- 포인트 클레이 색을 한 화면에 여러 군데 흩뿌리지 않는다. 가장 중요한 액션 하나에.

## 타이포그래피

- 기본 폰트: Pretendard (없으면 system-ui, sans-serif)
- 굵기는 두 가지만: 400(본문), 600(강조·제목). 700 이상은 쓰지 않는다.
- 크기 위계:

| 스타일 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| 제목 | 22px | 600 | 화면 대표 제목 |
| 섹션 제목 | 18px | 600 | 카테고리명, 카드 헤더 |
| 본문 | 15px | 400 | 기본 텍스트 |
| 보조 | 13px | 400 | 도메인, 저장 시각, 설명 |
| 캡션 | 12px | 400 | 뱃지, 작은 안내 문구 |

- 크기는 위 5단계만. 14, 16, 17 같은 중간값을 임의로 만들지 않는다.
- 위계는 크기와 색(본문/보조)으로 만들고, 굵기를 남발하지 않는다.

## 여백·모서리

- 모서리: 카드 12px, 버튼·입력창 8px 라운드.
- 여백은 넉넉하게. 요소가 답답하게 붙지 않도록.
- 세로 리듬은 rem 단위(1rem, 1.5rem, 2rem), 컴포넌트 내부 간격은 px(8, 12, 16).
- 그림자는 절제. 카드가 배경에서 살짝 떠 보이는 정도의 부드러운 그림자만.
  진한 그림자·글로우·네온 금지.

## 컴포넌트

### 버튼
- 주요 액션(저장 등): 배경 `#BE7250`, 텍스트 흰색, 8px 라운드. hover 시 `#A85E3E`.
- 보조 액션: 배경 흰색 또는 투명, 테두리 `#EDE9E3`, 텍스트 본문색.
- 한 화면에 클레이 채움 버튼은 하나만. 나머지는 보조 스타일로.

### 입력창
- 배경 흰색, 테두리 `#EDE9E3`, 8px 라운드.
- 포커스 시 테두리를 클레이 `#BE7250`로, 은은한 포커스 링.
- placeholder는 보조 텍스트색 `#8B857C`.

### 카드
- 배경 흰색, 테두리 `#EDE9E3`, 12px 라운드.
- 부드러운 그림자로 배경에서 살짝 띄운다.
- 링크 카드 구성: 썸네일 + 제목(본문) + 설명(보조) + 도메인(보조) + 저장 시각(캡션).

### 활성 항목 (사이드바 등)
- 선택된 카테고리는 배경 `#F5EBE4`(클레이 틴트) + 텍스트/아이콘 클레이색.
- 비선택 항목은 보조 텍스트색, 배경 없음.

## 아이콘

- 이모지 절대 사용 금지.
- lucide-react 라인 아이콘만 사용.
- 아이콘 색은 텍스트색을 따르되, 활성 상태에서만 클레이색.

## 레이아웃

- 좌측 사이드바(카테고리) + 중앙 콘텐츠(홈/피드) 구조.
- 모바일에서는 사이드바를 하단 탭 바 또는 드로어로 전환.
- 모바일 우선 반응형.

## 서랍 모티프 (홈 화면)

- 홈의 URL 입력 영역은 "서랍" 컨셉을 시각적으로 살린다.
- 서랍은 클레이색(`#BE7250`) 계열로 그리되, 진짜 갈색 나무가 아니라
  플랫하고 기하학적인(노션 로고 같은) 미니멀 벡터 스타일로.
- 나무색을 과하게 쓰지 않는다 — 클레이톤으로 "나무 느낌"만 은은하게.
